## HITL Pattern

The user feedback surface should be a tool-shaped interaction, not a fragile
format convention.

For palantir-mini:

- Ask focused questions.
- Prefer structured choices plus optional free text.
- Use progressive disclosure for deeper evidence.
- Keep the action space small and meaningful.

## Ontology Engineering Use

Use this pattern when asking the user to confirm:

- Meaning.
- Scope.
- Boundary.
- Risk.
- Evaluation criteria.
- Next routing action.
