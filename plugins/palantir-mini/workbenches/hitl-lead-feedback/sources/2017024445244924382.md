# Source Card: Playgrounds

## Direct Evidence

The direct X article describes standalone HTML playgrounds that let users
visualize a problem, interact with it, and copy an output prompt back into
Claude Code.

## HITL Pattern

A playground is useful when plain text is too narrow for the feedback needed.
The user should be able to:

- Adjust options.
- Approve, reject, or comment.
- Export a prompt, diff, or structured summary.
- Re-enter that result into the agent loop.

## Ontology Engineering Use

Use this pattern when the user must shape:

- Ontology object boundaries.
- Action semantics.
- Scenario buckets.
- Evaluation priorities.
- UI or route surfaces tied to ontology meaning.
