## HITL Pattern

Human feedback should be modeled as a stable interaction state, not as an
ad-hoc tool swap.

For palantir-mini, this means:

- Keep the workbench as a stable docs/skill surface.
- Use review cards and artifacts as state transitions.
- Do not add or remove runtime tools to represent each review phase.
- Defer optional details through progressive disclosure.

## Ontology Engineering Use

Use this pattern to keep HITL review predictable across:

- Plan mode.
- SIC fill/review.
- DTC boundary review.
- Router/delegation review.
- Compaction or handoff.
