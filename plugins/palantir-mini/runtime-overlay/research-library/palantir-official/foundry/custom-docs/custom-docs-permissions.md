# Permissions for custom documentation

You can use the Palantir platform's security features to manage permissions for your custom documentation and share it with others.

## Permissions to view custom documentation

Only users that have View access to a documentation code repository will be able to see the docs published by that repository. In other words, any users with view permissions on a custom documentation repository can see the published version of the documentation.

This enables you to publish docs to a limited audience, or iterate with a limited user group before a wider release. These access controls also reduce the risk of unwanted sharing across organizations or groups.

You can manage these permissions in the Palantir platform file system, based on the location of the code repository.

## Permissions to create custom documentation

All users can create repositories using the documentation template, but the ability to publish a repository as in-platform custom documentation requires an enrollment administrator to enable publishing.
