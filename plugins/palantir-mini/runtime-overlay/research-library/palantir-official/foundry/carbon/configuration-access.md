# Access configuration

Switch to the **Access** configuration tab (highlighted in red in the image below) to configure the [location](#workspace-location) and [visibility](#workspace-visibility) of a Carbon workspace, as well as [set a default workspace](#default-workspace) for a user group.

<img src="./media/configure-workspace-access-tab.png" alt="Access tab" width="300" />

### Workspace location

The **Location** section contains a link to the resource containing your workspace (highlighted in red in the image below). Clicking this link will allow you to change access settings or move the file in a Project.

<img src="./media/configure-workspace-location.png" alt="Location" width="300" />

### Workspace visibility

In the workspace visibility section (highlighted in red below), [Carbon Organization Administrators](/docs/foundry/carbon/permissions-configure/#configure-administrator-permissions) can set workspaces either as **Link only** or **Promoted**. A link-only workspace is only accessible by going directly to the URL of the workspace or opening it from a Project. A promoted workspace appears in the [Navigation Menu](/docs/foundry/carbon/workspaces-navigation/) for all members and guests of an organization.

<img src="./media/configure-workspace-workspace-visibility.png" alt="Workspace visibility" width="300" />

### Default workspace

Users can have access to more than one Carbon workspace. Thus, we recommend setting a default workspace for each user group. You can do so in the **Default workspace** section in the **Access** tab (highlighted in red below). Follow the [instructions on how to set a default workspace](/docs/foundry/carbon/permissions-default-workspace/).

<img src="./media/configure-workspace-default-workspace.png" alt="Set default workspace" width="300" />
