# Example workspaces

## Aviation Workspace

Carbon workspaces are intended to provided a streamlined Foundry interface for operational users. The screenshot below depicts a simple workspace designed for an airline analyst to review alerts related to the performance of particular flight routes. This Carbon workspace includes quick links to relevant applications and data, along with access to the analyst's "favorite" individual objects. To learn how this workspace was constructed, see the [getting started tutorial](/docs/foundry/carbon/getting-started/).

![Example Aviation workspace](/docs/resources/foundry/carbon/getting-started-end-state.png)

## Claim Portal

Carbon workspaces also allow users to understand their data and make decisions as part of operational workflows. For example, a company may want to create an interface for customer support analysts to investigate warranty claims. In the Carbon workspace depicted below, users can manage, triage, and action claims made to the customer support team.

![Example Claim Portal workspace](/docs/resources/foundry/carbon/carbon-workspace.png)

Carbon workspaces are usually created and edited in a graphical user interface (as shown in the [getting started tutorial](/docs/foundry/carbon/getting-started/)), but you can also edit the YAML file specifying a Carbon workspace directly. To see the YAML for the Claim Portal workspace, examine the [YAML code example](/docs/foundry/carbon/code-example/) and the [YAML configuration reference](/docs/foundry/carbon/code-reference/).
