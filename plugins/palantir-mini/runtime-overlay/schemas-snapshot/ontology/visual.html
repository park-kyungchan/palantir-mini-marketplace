Domain Architecture
Three semantic domains with unidirectional dependency: DATA (ground truth) → LOGIC (reasoning) → ACTION (execution). Security is a governance overlay, not a semantic domain.
DATA — Ground Truth

What EXISTS right now

"Does this describe WHAT EXISTS right now — independent of interpretation or action?"
A library's catalog: records what books exist, their titles, ISBNs, page counts, shelf locations — facts independent of how anyone uses them.

Dependency Rules:
DATA reads nothing (foundational, standalone)
LOGIC reads ObjectType, Property, ValueType from DATA
ACTION reads ObjectType, Property + Function, LinkType
SECURITY is a governance overlay — cross-cuts all three domains

Hard Constraints

Consistency Invariants
8 cross-domain verification rules that guarantee the ontology schema is internally consistent. These are checked by semantics.test.ts (43 tests, 248 assertions).
Transition Zones
