# Rule 12 — Lead Protocol + Operations

Governs Lead in Ontology-Driven multi-teammate sessions. Consolidation hub absorbing rules 06 (agent-teams), 13 (task-granularity), 14 (session-duration-replacement), 15 (briefing-template) per harness-base-mode blueprint §12 license (2026-04-29). Lead-direct (§Team default) remains default for single-agent work. External evidence: Anthropic Prithvi 2026-03-24 harness-design post + DevCon 5 AI-FDE pattern.

## §Model policy

- Fixed by role: `researcher` / `docs-researcher` / `ontologist` / `evaluator` → `opus`; all implementers → `sonnet`. Lead is `opus[1m]`. Implementer needing Opus reasoning → delegate to `researcher`, not bump model.
- Lead MUST NOT pass `model` to `Agent(...)`, `SendMessage`, or `TaskCreate`. Frontmatter `model:` is SSoT.

## §3-tier model fallback chain (v3.9.0)

Lead model is pinned via settings.json. Default `opus[1m]`. Fallback chain (in order):
- Tier-1 (primary): `opus[1m]` (1M context)
- Tier-2: `opus` (200K context)
- Tier-3: `sonnet[1m]` (1M context, smaller model)
- Tier-4: `sonnet` (200K context, smaller model)

`lead-model-availability-check` SessionStart hook probes primary availability. If unavailable: emit `lead_model_deprecation_warning` event with `withWhat.refinementTarget = { kind: "rule-12-fallback-chain" }` + `payload.suggestedFallback = <next-tier>`. Operator action: edit settings.json to next tier OR set `LEAD_MODEL_OVERRIDE=<model-id>` (audited via `lead_model_override_invoked` event). Emergency override env bypasses the chain entirely.

## §Team default + Lazy-spawn (absorbs 6)

- Lead-direct is the default. Use teammates only when the user explicitly requests team or parallel execution.
- Keep file ownership disjoint and verify teammate output before synthesis.
- Put stable context in shared docs, hooks, and tests — not in teammate memory.
- Use hooks and audits for enforcement instead of relying on chat state alone.
- Do NOT pre-spawn teammates. Spawn when `blockedBy` clears; shut down on `TaskCompleted`. Re-spawn beats idle polling — fresh context, no stale state, no idle-notification loop.
- `idleCount >= 3 && blockedByDepth > 1` → auto `shutdown_request`. One-level-blocked teammates stay warm.

## §Lead-direct harness wrapping (v3.1.0)

- In tracked palantir-mini project (`.palantir-mini/harness/` exists), Lead-direct work MUST run inside a bound SprintContract.
- If no contract found: invoke `/palantir-mini:pm-quick-sprint "<brief>" <scope>` BEFORE first non-trivial Edit/Write/MultiEdit. SessionStart `harness-base-mode-advisory` hook auto-bootstraps a default Quick Sprint when absent (plugin v3.12.0+; D-tier auto-bootstrap).
- Bypass: `PALANTIR_MINI_HARNESS_BYPASS=1` (audited) or user says "skip sprint-harness".
- Quick Sprint suffices for hot-fixes / docs / single-file change. Multi-file feature work → escalate to `mode="full"` via `/palantir-mini:pm-harness-plan` + `pm-harness-sprint`.

## §Pre-delegation analysis framework (v3.4.0)

Lead is intent-interpreter + infra-caller + recipe-reviewer + spawn-approver — NOT direct executor. For any task touching ≥3 files, Lead MUST call `/palantir-mini:pm-delegate-or-direct` BEFORE first Edit/Write/MultiEdit.

Lead 4-step orchestration: (1) intent 해석; (2) infra skill 호출 (`/pm-delegate-or-direct`); (3) recipe 검토 + 조정; (4) sprint contract bind + Agent spawn.

Infra 9-field recipe (auto-generated by skill): agent / mcp_tools / skills / sprint_args / critical_files / verify_command / memory_layers / out_of_scope / briefing_template (rule 12 §Briefing 5-section auto-fill).

Sonnet subagent constraints: do NOT decide tool selection / agent selection / skill selection / scope changes / extra agent spawn. Execute literal instructions only.

Cross-ref: rule 16 §Quick Sprint, rule 26 §Axis E, rule 07 §Agent file-ownership.

## §Lead self-test directive (v3.8.0)

- Lead MUST invoke `delegate_or_direct` MCP at least once per non-trivial sprint (multi-file or new-feature work). This dogfoods the pre-delegation framework and emits the `delegation_recipe_generated` event that satisfies `pre-delegation-check` bypass.
- "Non-trivial sprint" = any sprint with complexityHint ≠ "trivial" OR scopePaths count ≥ 2.
- Invocation may be at sprint-start (recommended) or at the first task that crosses ≥3 files threshold.
- `delegation_recipe_generated` event count per sprint MUST be ≥ 1 when Lead-direct edits ≥ 3. Pre-delegation-check reads this count; 0 invocations = pre-delegation framework is dead code (architecture review R3-F2 / B4 root cause).
- Bypass: `PALANTIR_MINI_PREDELEGATION_BYPASS=1` (audited) — records that dogfooding was consciously skipped.

## §MCP-First protocol (v3.10.0)

- Lead MUST consult MCP impact tools (`impact_query` / `semantic_change_plan` / `pm_workflow_lineage_query` / `pm-impact-quick`) BEFORE first Edit/Write/MultiEdit on any task touching ≥1 file. File reads only for MCP-flagged uncertainties or read-before-edit context.
- Target: ≥80% MCP-first compliance ratio per session, measured by `lead-token-audit.ts` from `lead_mcp_first_compliance:passed` vs `:bypassed` events.
- `pre-edit-impact-mcp-first` hook (PostToolUse:Edit, advisory only) emits compliance events. Bypass: `PALANTIR_MINI_MCP_FIRST_BYPASS=1` (audited).
- pm-recap §MCP-First Compliance section reports ratio + estimated tokens saved (vs file-read baseline ~25K/question).
- Rationale (sprint-060 W4 user directive): Lead's default file-read pattern can burn 50-200K tokens per impact analysis. palantir-mini Brain-of-Swarms is nullified at the moment of greatest value if MCP-first not enforced. Sprint-060 measured 60-65% of 1M context burned; without protocol, sprint-061 might not fit.

## §Phase-gate contract

- State-file validation fires on `SubagentStop` (Blocking), NOT `PostToolUse`.
- Each agent `.md` declares output contract: file path + required fields + shape. Hook validates; exit-2 on mismatch.

## §Subagent decision audit invariant (v3.4.0)

Every spawned subagent's MCP tool calls (apply_edit_function / commit_edits / emit_event) auto-emit `agent_decision_logged` event with `{toolName, agentName, inputDigest, reasoning, hypothesis, sprintRef, memoryLayers, correlationId}` payload (W1.E hook). Every Agent tool spawn captured with full prompt + correlationId (W1.G hook). Lead query interface: `/palantir-mini:pm-agent-audit-trail` (W1.H skill).

## §Agent frontmatter standard

- Required: `name`, `description`, `tools`, `model`. Recommended: `maxTurns`, `memory`.
- Forbidden: `initialPrompt` (bypasses Lead briefing + blocks subagent_type auto-resolve).
- `SessionStart` hook rejects non-conformant (exit-2).

## §Research-over-codegen

Lead (Opus) produces blueprints + curated references + implementation guidance with cited examples. Implementers (Sonnet) do the writing. Lead orchestrates + reviews.

## §Task granularity (absorbs 13)

- Each task description must include 4 sections: **DELETE** (what to remove), **ADD** (what to create), **KEEP** (what NOT to touch), **VERIFY** (shell command).
- Each task owns exactly 1 primary file path. Barrel/index re-exports count as part of the owning split; multi-file tasks require explicit barrel/index exception in description.
- Context budget ≤ 15K tokens per task (prompt + shared context + expected output). Lead estimates during TaskCreate; >12K triggers split review.
- `blockedBy` must be explicit — no implicit ordering.
- Per-teammate `in_progress` cap = 2 concurrent tasks (prevents claim-hoarding).
- VERIFY step must be a concrete shell command (e.g. `bunx tsc --noEmit`, `bun test --filter X`, `grep -c "pattern" file`). Task completion requires VERIFY green before `TaskUpdate(completed)`.

## §Session lifecycle (absorbs 14)

- At **3h** session mark, Lead broadcasts: "Terminate when task complete; no new spawns; prepare `/palantir-mini:pm-ship`."
- At **4h**, `Agent` spawn is blocked by `session-duration-alarm` hook.
- If a teammate has 5+ consecutive unread Lead DMs, auto-spawn replacement via `auto_spawn_replacement` MCP; transfer owned tasks; kill original.
- Submodule rebases (multi-commit cherry-picks with likely semantic divergence) are Lead-only. Teammates do NOT attempt.
- Cross-project ship/PR/merge orchestration is Lead-only.
- **Context-overflow re-spawn (sprint-005 learning #6, 2026-04-30)**: at iteration boundary, if prior agent's token usage > 80K, auto-spawn replacement via `auto_spawn_replacement` MCP — fresh context, no stale state. This is in addition to the 5+ idle DMs trigger (which fires within an iteration for unresponsive agents). Two triggers, distinct semantics: 5+ idle DMs = mid-iteration unresponsiveness; >80K tokens = inter-iteration context bloat.

## §Briefing template (absorbs 15; v3.3.0+ 5-section)

Every `Agent(...)` spawn prompt must include 5 sections (v4.1.0 plugin + rule 26 v1.0.0):

1. **Speed target** — explicit time-per-task estimate (e.g. "5-10min per primitive").
2. **Claim order** — numbered list of task IDs to claim in sequence.
3. **No idle polling directive** — "Work continuously until all claimed tasks done. Then self-shutdown. Do NOT send idle notifications between tasks."
4. **Reply-in-text expectation** — "Respond to Lead DMs with plain text status when asked. Not just idle notifications."
5. **Memory layer declaration** — which of 4 agentic memory layers (working / episodic / semantic / procedural) the work refines, with one-line rationale per layer (rule 26 v1.0.0 §Axis E).

`briefing-template-validate` hook (SubagentStart):
- Sections 1-4 — Blocking; exit-2 on missing.
- Section 5 — Blocking by default as of 2026-05-08 (default-flip PR landed sprint-046). Emergency bypass: `PALANTIR_MINI_BRIEFING_5SEC_ADVISORY_ONLY=1` (audited via `briefing_advisory_only_invoked` event).

## §Complex-task EnterPlanMode protocol (v3.5.0)

For prompts touching ≥3 files OR introducing a new feature OR proposing architectural change, Lead MUST: (1) call `/palantir-mini:pm-delegate-or-direct` (rule 12 §Pre-delegation); (2) enter plan mode via `/plan`; (3) author plan with phases + diagrams + verification + scope; (4) await user approval before first non-trivial Edit/Write/MultiEdit.

`complex-task-detector` UserPromptSubmit hook surfaces advisory when heuristic matches (length ≥ 800 OR ≥2 keywords OR ≥3 file paths). Bypass: prompt contains `skip-delegate` literal.

## §Harness Engineering context awareness (v3.5.0)

`harness-engineering-context-loader` SessionStart hook auto-injects ~30-line excerpts from 4 1차 자료 (scaling-managed-agents / harness-design / 4-vendor / opus-4-7-postmortem) as `additionalContext` so Lead's first turn already knows: Brain/Hands/Session decoupling, Prithvi 3-agent pattern, 4-vendor convergence, Opus 4.7 token inflation. palantir-mini Brain dispatches across 7 species (rule 16 v4.0.0 §0); Lead must NOT default to Lead-direct one-shot for non-trivial work.

## §Hook v2 conventions (absorbs 6 §Hook field v2)

- `timeout` is seconds (not ms). `statusMessage` on blocking hooks conveys what they're doing.
- Advisory hooks mark `async: true`. PreToolUse/PostToolUse use `if` (permission-rule format) to avoid unmatched hook spawns.
- Structured JSON output supports `additionalContext`, `hookSpecificOutput.permissionDecision`, `continue/stopReason` — prefer over stdout hacks.
- `once: true` (skills only) runs a skill exactly once per session.

## §Pre-delegation hard gate (v3.6.0)

`pre-delegation-check.ts` PreToolUse:Edit|Write|MultiEdit hook codifies §Pre-delegation analysis framework as a hard gate. When cumulative session edits ≥ 3 AND no `delegation_recipe_generated` event in last 30 min AND active SprintContract.mode != "full" → block with `permissionDecision:"deny"`. Synthesis paths exempt (`~/.claude/plans/**`, `BROWSE.md`, `INDEX.md`). Bypass: `PALANTIR_MINI_PREDELEGATION_BYPASS=1` (audited).

Per-sprint counter (`lead-direct-edit-watch.ts` v3.6.0+) tracks per-SprintContract edit count alongside cumulative session count; per-sprint advisory@3 / block@5; harder gate wins.

Marker escalation: `complex-task-detector` writes `.complex-task-pending.json` when ≥2-of-3 heuristic conditions matched; `pre-delegation-check` escalates block reason if marker present.

## §Lead-direct edit threshold (v3.4.0)

Single sprint context: ≤3 files Lead-direct OK; >3 files MUST `/palantir-mini:pm-delegate-or-direct` first.

Cumulative session: advisory at 5 (`lead-direct-edit-watch` hook PostToolUse on Edit|Write|MultiEdit); blocking at 15. Bypass `PALANTIR_MINI_LEAD_DIRECT_BYPASS=1` (audited via `lead_direct_bypass_invoked` event).

Synthesis paths exemption: `~/.claude/plans/**` + `~/.claude/research/{BROWSE,INDEX}.md` exempt from blocking gate.

## §Task context budget hard gate (v3.7.0)

`task-context-budget-enforcer.ts` PreToolUse:Agent hook codifies §Task granularity context-budget cap (≤15K tokens) as a hard gate. Estimates briefing prompt tokens (chars / 3.5 conservative). Advisory at 10K tokens; blocking at 15K tokens. Bypass: `PALANTIR_MINI_TASK_BUDGET_BYPASS=1` (audited via `task_budget_bypass_invoked` event). Pairs with `briefing-template-validate` (structural) + `pre-delegation-check` (gate-on-edit) — this hook checks SIZE pre-spawn.

Rationale (sprint-057 W6 driver, user explicit ask 2026-05-07): post sprint-057 W1 reverted 22 plugin agents from [1m] (1M ctx) to standard 200K — finer task splitting required to avoid 200K exhaustion. "Lead가 tasks를 더 분할하도록 hooks 등을 강제해야겠다" — hook enforces split before spawn.
