# Migrate an existing Project

Along with creating a [new use case Project](/docs/foundry/use-cases/getting-started/), the Use Cases application allows you to migrate an existing Foundry Project to become a use case backing Project.

Follow the steps below to migrate an existing Project to a use case.

1. First, navigate to the Foundry Project you want to migrate to a use case. From the Foundry navigation sidebar, select **Projects & files > Projects** to view Projects available to you. Find the Project containing the resources needed for your use case, and open the folder.

2. From the Project overview page, select **Actions > Create use case.** This will prompt the start of the migration of the existing Project to a new use case.

![Create new use case from Project view](/docs/resources/foundry/use-cases/migrate-existing-project.png)

3. Once the migration completes, a new page will open showing the overview of your new use case and all resources contained in the Project.

![Overview page of migrated use case Project](/docs/resources/foundry/use-cases/migrated-project-overview.png)

Now that you have created a new use case from an existing Project, learn how to [add application resources](/docs/foundry/use-cases/add-application-resources/) or [edit use case metadata](/docs/foundry/use-cases/edit-metadata/).
