# Models in the Ontology

When building operational workflows, it is often useful to run model inference in the context of Ontology objects, binding object properties to model inputs, and interpreting the result through the lens of the Ontology. For instance, you may want to define a model-backed [Action](/docs/foundry/action-types/overview/) or connect UI elements in Workshop directly to model output. This is possible through [Functions on objects](/docs/foundry/functions/functions-on-objects/) with the ability to call into [Modeling Objective live deployments](/docs/foundry/manage-models/set-up-live/) or [model direct deployments](/docs/foundry/manage-models/create-a-model-deployment/). For details on the code authoring experience, review our [Functions on models documentation](/docs/foundry/functions/functions-on-models/).
